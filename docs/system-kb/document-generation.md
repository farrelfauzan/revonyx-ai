# Document Generation Guidelines

When a user asks you to generate a document in a specific format (PDF, Word/DOCX, Excel/XLSX), follow these guidelines to produce the best possible output.

## General Rules

- Always write your response in clean, well-structured Markdown
- Use proper heading hierarchy: `#` for the document title, `##` for major sections, `###` for subsections
- Write complete, professional-quality content — never use placeholder text like "[insert here]" or "TBD"
- Do NOT mention the file format or conversion process in your response
- Do NOT wrap the entire output in a code block
- Focus on delivering substantive, actionable content

## Document Structure Best Practices

### For Reports and PRDs
- Start with a title and brief executive summary
- Organize into logical sections with clear headings
- Use bullet points for lists of items, features, or requirements
- Use numbered lists for sequential steps or ranked items
- Include tables for comparative data or structured information
- End with conclusions, recommendations, or next steps

### For Tabular/Data Documents
- Use Markdown tables with clear column headers
- Keep data organized and consistently formatted
- Use separate tables for different datasets, each with its own heading
- Include summary rows or totals where appropriate

### For Business Documents
- Maintain a professional tone throughout
- Include relevant dates, version numbers, or document metadata in the content
- Structure content for easy scanning with headers and bullet points
- Use bold for key terms and italic for emphasis

## Formatting Tips

- **Bold** for important terms, names, and key concepts
- *Italic* for emphasis, definitions, and notes
- `Code` for technical terms, file names, commands, or code snippets
- Use horizontal rules (`---`) to separate major document sections
- Keep paragraphs concise — prefer 3-5 sentences per paragraph
- Use consistent formatting throughout the document
